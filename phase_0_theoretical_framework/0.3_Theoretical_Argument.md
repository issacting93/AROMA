# 0.3 The Theoretical Argument

The field of AI-mediated mental health support currently suffers from a fundamental conceptual conflation: it routinely confuses *Support Type* (what is being given) with *Care Role* (who the system is being).

Existing foundational taxonomies, such as Cutrona & Suhr's (1992) Social Support Behavior Code, excel at defining support types (Emotional, Informational, Esteem, Network, Tangible) but contain no mechanism for defining the relational stance of the provider. Consequently, when these frameworks are applied to AI, the resulting systems default to a static execution of support categories—often informational or cognitive-behavioral—ignoring the relational identity required to deliver that support effectively. 

Conversely, when clinical AI role taxonomies are proposed (e.g., Vaidyam et al., 2019; Stade et al., 2024), they categorize systems by clinical purpose (e.g., Assessor, Therapist-Adjunct) rather than by interactional dynamic. These roles are treated as mutually exclusive, system-level assignments. Current AI mental health systems reflect this gap: they are profoundly *role-locked*. Woebot is permanently a CBT Coach; Replika is permanently a Companion. 

Yet, as human-human peer support literature demonstrates (e.g., Sharma et al., 2020), effective caregiving is not static. Human supporters fluidly transition between roles within a single interaction—shifting from an empathic Listener when the user is venting, to a Reflective Partner when the user is processing, to an Advisor when the user requests guidance. This dynamic shifting is absent in both current AI systems and the frameworks used to design them. Furthermore, prior work lacks a formalized *Core Function* layer that explicitly links a chosen role to its intended psychological outcome, and entirely ignores *Interaction Modality* as a structural constraint on role viability.

The Parsonian "sick role" illustrates why this missing role abstraction is critical: occupying a caregiving role presupposes distinct expectations from the provider. When an AI system assumes an authoritative role (like Advisor) without the corresponding agency to enact real-world change, it invites catastrophic failure modes, most notably the *Therapeutic Misconception* (Khawaja & Bélisle-Pipon, 2023). Users project clinical authority onto the system, only to be abandoned when the system structurally cannot intervene.

Therefore, a multi-dimensional role ontology is urgently needed. AROMA provides the first empirically grounded taxonomy that separates Support Type (D1) from Care Role (D2), linking them via a Core Function (D3) that is enacted through explicit conversational Support Strategies (D4), all constrained by Interaction Modality (D5). This framework provides the analytical leverage to design systems that can detect when a role transition is warranted and execute it fluidly, mirroring the dynamic reality of human care while avoiding the ethical traps of unearned AI authority.
