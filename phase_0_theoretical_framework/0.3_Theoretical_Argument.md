# 0.3 The Theoretical Argument

The field of AI-mediated mental health support currently suffers from a fundamental conceptual conflation: it routinely confuses *Support Type* (what is being given) with *Care Role* (who the system is being).

Existing foundational taxonomies, such as Cutrona & Suhr's (1992) Social Support Behavior Code, excel at defining support types (Emotional, Informational, Esteem, Network, Tangible) but contain no mechanism for defining the relational stance of the provider. Consequently, when these frameworks are applied to AI, the resulting systems default to a static execution of support categories—often informational or cognitive-behavioral—ignoring the relational identity required to deliver that support effectively.

Conversely, when clinical AI role taxonomies are proposed (e.g., Vaidyam et al., 2019; Stade et al., 2024)[^1], they categorize systems by clinical purpose (e.g., Assessor, Therapist-Adjunct) rather than by interactional dynamic. These roles are treated as mutually exclusive, system-level assignments. Current AI mental health systems reflect this gap: they are profoundly *role-locked*. Woebot is permanently a CBT Coach; Replika is permanently a Companion.

**Effective caregiving is not static.** People take on different roles within a single interaction — shifting from an empathic Listener when the user is venting, to a Reflective Partner when the user is processing, to an Advisor when the user requests guidance. 

This fluidity is grounded in Blumer's (1969) concept of *role-taking*: the mechanism by which an actor reads the other's expressed state, takes their perspective, and calibrates their own stance accordingly. *A role is not declared in a single utterance — it is enacted across a sequence of turns through this read-and-respond cycle.* 

AROMA formalizes this dynamic by extending the **Integrated Model of Advice-giving (IMA)** (Feng, 2009). While the IMA identifies sequential moves (empathy → inquiry → advice) within a single role context, AROMA scales this logic into a multi-role taxonomy applicable across all social support stages.

When an AI system assumes an authoritative role (like Advisor) without the corresponding agency to enact real-world change, it creates a structural **Authority-Agency Paradox**. 

Drawing on Parsons (1951), AROMA argues that this paradox is fundamentally an **obligation gap**: the AI receives authority without the accountability of follow-through, while the user receives support without reciprocal commitment. This dissolves the **structural binding**—the mutual expectations—that makes human care relationships therapeutic rather than merely comforting. This "unearned authority" can lead to a *therapeutic misconception*, where the user incorrectly believes they are in a governed, accountable care relationship.

Therefore, a multi-dimensional role ontology is urgently needed. AROMA provides the first theoretically grounded taxonomy, validated through empirical annotation, that separates Support Type (D1) from Care Role (D2), linking them via a Core Function (D3) that is enacted through explicit conversational Support Strategies (D4), all constrained by Interaction Modality (D5). This framework provides the analytical leverage to design systems that can detect when a role transition is warranted and execute it fluidly, mirroring the dynamic reality of human care while avoiding the ethical traps of unearned AI authority.
